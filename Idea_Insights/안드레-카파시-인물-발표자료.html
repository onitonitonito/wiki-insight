안드레 카파시

AI 패러다임의 나침반이자 사상가

학문적 뿌리부터 OpenAI 창립과 테슬라 오토파일럿 총괄, 유레카 랩스 창업을 거쳐 2026년 5월 앤스로픽(Anthropic) 합류까지. 그가 주창한 Software 3.0과 에이전틱 엔지니어링 시대의 생존 전략을 분석합니다.

ACADEMIC FOUNDATIONS
02 / 11

학문적 뿌리와 CS231n

글로벌 엘리트 코스
1986년 슬로바키아 출생 후 토론토로 이주. 토론토 대학교에서 컴퓨터과학 및 물리학 학사(2009)를 마친 후, 브리티시 컬럼비아 대학교(UBC)에서 컴퓨터과학 석사(2011)를 취득하고 스탠퍼드 대학교 컴퓨터비전 박사(2015) 학위를 받았습니다.
스탠퍼드 CS231n의 탄생
컴퓨터 비전의 대모 '페이페이 리(Fei-Fei Li)' 교수의 지도 하에 최초로 스탠퍼드 딥러닝 강의인 CS231n을 설계하고 가르쳤습니다. 이는 현재 전 세계 딥러닝 및 합성곱 신경망(CNN) 교육의 글로벌 표준이 되었습니다.
CAREER TRAJECTORY
03 / 11

테슬라에서 앤스로픽(Anthropic)까지

OpenAI & Tesla 디렉터
2015년 OpenAI의 창립 연구 멤버로 연구를 주도했으며, 2017년 테슬라(Tesla)로 이직하여 AI 디렉터로서 자율주행 오토파일럿 컴퓨터 비전 시스템과 거대 '데이터 엔진' 루프를 설계·총괄했습니다.
유레카 랩스 창업 (2024)
2023년 OpenAI 복귀 후 GPT-4 프로젝트에 기여한 뒤, 2024년 7월 AI 네이티브 학교 '유레카 랩스(Eureka Labs)'를 창업하여 1:1 AI 조교와 인간 교사가 공생하는 교육 패러다임(LLM101n 코스)을 제시했습니다.
앤스로픽 합류 (2026.05)
2026년 5월, 앤스로픽(Anthropic)에 전격 합류했습니다. 현재 사전학습(Pretraining) 팀에서 차세대 클로드(Claude) 핵심 기초 모델의 학습 알고리즘 고도화를 연구 중입니다.
SOFTWARE PARADIGM SHIFT
04 / 11

소프트웨어 패러다임의 진화

Software 1.0
인간이 직접 규칙과 조건식(If-else), 알고리즘의 모든 라인을 키보드로 직접 명령하여 작성하는 전통적인 결정론적 프로그래밍 패러다임.
Software 2.0 (2017)
카파시가 2017년 정의한 개념. 인간이 목표(손실 함수)와 데이터만 제시하고, 최적화 알고리즘이 신경망의 '가중치(Weight)'라는 새로운 코드를 스스로 생성하는 패러다임.
Software 3.0 (2025)
인간이 자연어로 "목적과 의도(Vibe)"만 표현하면, 지능형 에이전트(Claude Code 등)가 컨텍스트 윈도우를 제어 레버 삼아 스스로 코드를 컴파일하고 작성하는 프롬프트 기반 코딩.
VIBE CODING ERA
05 / 11

바이브 코딩과 격차의 확대

"나는 더 이상 코드를 짜지 않는다"
2025년 초, 카파시는 코딩 에이전트(Cursor, Claude Code)의 비약적 발전에 힘입어 자연어로 아이디어와 느낌(Vibe)을 주면 코딩은 에이전트가 완수하는 '바이브 코딩(Vibe Coding)' 트렌드를 개막하며 수작업 코딩 종말을 고했습니다.
양극화되는 AI 격차 (The Growing Gap)
2026년 4월 카파시는 AI를 단순 챗봇/검색기로 취급하는 대중과, 에이전트·MCP·커스텀 스크립트를 조율하여 1:N 복리 자동화 업무망을 만드는 파워 유저 사이의 생산성 격차가 기하급수적으로 벌어지고 있음을 경고했습니다.
COGNITIVE CODE RECONSTRUCTION
06 / 11

에이전트 제어를 위한 카파시 코딩 원칙

01

Think Before Coding (코딩 전 사유)

불확실성에 기반해 억측 코딩하지 않고, 모호함을 명시적으로 드러내어 사용자에게 트레이드오프 질문.

02

Simplicity First (단순함 최우선)

오버엔지니어링, 과도한 예외 처리, 불필요한 설정 추가를 금지하고 200줄의 복잡한 구조를 50줄로 다이어트.

03

Surgical Changes (외과 수술식 변경)

주변의 불필요한 포맷이나 스타일을 함부로 교정하지 않고, 오직 요구된 핵심 영역만 정확하고 날카롭게 수정.

04

Goal-Driven Execution (목표 주도 검증)

막연한 요구 대신 단계별 성공 기준을 스스로 정의하고, 단계마다 검증 테스트 루프를 거쳐 전진.

결정론적 한계 보완 스킬
이 지침은 AI 에이전트가 함부로 넘겨짚고 API를 난잡하게 오버엔지니어링하여 시스템 부채를 증가시키는 고유 결함을 억제하도록 돕는 본 위키(Antigravity)의 핵심 룰셋입니다.
LLM WIKI METHODOLOGY
07 / 11

지식의 복리 자산화: LLM Wiki

1층: Raw-Data
수집된 기사, 회의록, 유튜브 대본, 이미지 등 정제되지 않은 상태의 불변 데이터(Immutable Source of Truth) 보관 레이어.
2층: Wiki-Pages
AI가 Raw-Data를 읽고 지식을 컴파일하여 나무위키 형태로 가공, 상호 교차 참조 및 백링크가 연결되어 지속적으로 디벨롭되는 축적의 위키 문서.
3층: Schema (AGENTS)
위키가 구조화되는 규격과 룰셋(AGENTS.md)을 제공하여 AI를 제멋대로 떠드는 챗봇이 아닌, 정교하고 통제된 위키 관리자로 정의하는 핵심 스키마.
SEQUOIA AI ASCENT 2026

사유와 이해의 분리

"Thinking can be outsourced, Understanding cannot."
"정보를 처리하고 유추하는 '사유(Thinking)'의 하위 연산은 에이전트에 아웃소싱해도 된다. 그러나 시스템 전체 아키텍처와 비즈니스 본질에 대한 '이해(Understanding)'는 오직 인간만의 영역이며, 결코 위임해서는 안 된다."
동물이 아닌 '유령(Ghosts)'을 제어하라
LLM은 본능이 있는 동물이 아닙니다. 강화학습(RL)에 의해 동작하며, 오직 채점 가능(수학, 컴파일)한 영역에서만 천재성을 띠고 상식 영역에선 들쭉날쭉한 역량(Jagged Intelligence)을 보입니다. 감정적 프롬프트가 아닌 구조적이고 회로적인 한계를 통제해야 합니다.
ON-DEMAND LEARNING PROOF
09 / 11

온디맨드 학습과 탑다운 실증

카파시의 온디맨드 학습 조언
카파시는 2020년 트윗을 통해 "기초부터 넓게 배우는 Bottom-up을 피하고, 구체적인 프로젝트를 잡아 깊이 파고들며 필요한 것만 그때그때 배우라(Learning on-demand)"는 학습 원칙을 선언했습니다. 당시에는 1:1 튜터가 필수적이어서 실행이 어려웠던 이상론이었습니다.
가브리엘 피터슨의 탑다운 실증
이 조언을 챗지피티(ChatGPT) 튜터를 통해 '탑다운(Top-down) 학습법'으로 명명하고 실천하여 고졸 학력으로 OpenAI 연구원이 된 인물이 가브리엘 피터슨입니다. 그는 완성형 코드를 복사하는 '바이브 코딩'을 지양하고 작동 원리를 역추적 질문하며 기초를 완벽히 장악했습니다.
CONCLUSION
10 / 11

인간에게 남은 유일한 가치

"어떻게(How) 구현하고 작성할 것인가"의 가치는 지능형 에이전트가 가져갔습니다.
이제 인간은 "무엇을(What) 만들 것이며, 왜(Why) 해야 하는가"에 대해 답해야 합니다.
AI 시대에 마지막까지 대체되지 않고 살아남는 차별점은 오직 인간만의 독자적인 취향, 실존적 결단(의지), 그리고 결과에 대한 책임뿐입니다.
