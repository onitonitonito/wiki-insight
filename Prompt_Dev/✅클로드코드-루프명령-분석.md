> [!IMPORTANT]
> **AI 신사업 기회 발굴 및 자동화 관점에서의 핵심 takeaway**
> Claude Code의 `/loop` 명령어는 세션 기반의 **주기적 백그라운드 워커(Watchdog)** 역할을 수행합니다. 특정 목표 달성이 목적이 아닌 '지속적인 상태 모니터링 및 알림'에 최적화되어 있으며, 이는 개발자의 인지적 부하(Cognitive Load)를 줄이고 완전 자율형 작업 환경을 구축하는 데 필수적인 기능입니다. 우리 위키의 오토메이션을 고도화할 때 매우 중요한 멘탈 모델을 제공합니다.

## 1. `/loop` 명령어 개요
`/loop`는 활성화된 Claude Code 세션 내에서 사용자가 지정한 간격으로 특정 프롬프트나 명령어를 반복 실행하는 스케줄러 기능입니다.

- **기본 문법**: `/loop [주기] [프롬프트/명령어]`
- **주기 설정**: `s`(초), `m`(분), `h`(시간), `d`(일) 단위를 지원하며, 생략 시 기본값은 보통 10분입니다.

## 2. 핵심 특징 및 제약 사항
- **세션 종속성 (Session-Scoped)**: 루프는 현재 열려 있는 세션에 종속됩니다. 터미널(세션)을 종료하면 루프도 멈추며, 백그라운드 서버에서 영구적으로 도는 것은 아닙니다.
- **독립적 반복 (Autonomous Iteration)**: 매 주기마다 사용자의 승인 없이 지정된 명령을 자율적으로 실행하고 결과를 보고합니다. 이전 루프의 상태를 '기억'하지 않는 독립적인 실행(Stateless)이 기본입니다.

## 3. 대표적인 비즈니스 및 개발 활용 사례 (Use Cases)

### 🛠️ 개발 및 배포 모니터링 (Watchdog)
- **CI/CD 파이프라인 감시**: `/loop 5m 배포 상태를 확인하고 실패(Fail) 로그가 발견되면 즉시 요약해서 알려줘.`
- **PR(Pull Request) 상태 확인**: `/loop 30m 내가 올린 PR들의 리뷰 상태나 머지 여부를 확인하고 브리핑해줘.`
- **장기 테스트 모니터링**: 테스트 스위트가 도는 동안 백그라운드에서 결과를 주기적으로 체크.

### 📊 정보 수집 및 리서치
- **정기 뉴스/동향 스크랩**: `/loop 1h 최신 AI 관련 뉴스를 검색해서 주요 변화를 3줄로 요약해줘.`
- **문서 동기화**: 코드베이스 변경 사항을 감지하여 정기적으로 `README.md`나 문서 업데이트.

## 4. ⚖️ `/loop` vs `/goal`의 차이점
AI 에이전트를 다룰 때 이 두 가지 명령어의 용도를 명확히 구분하는 것이 중요합니다.

| 구분 | `/loop` (루프) | `/goal` (목표) |
|---|---|---|
| **실행 방식** | **고정된 시간 간격**으로 무한 반복 | **조건(목표)**이 달성될 때까지 연속 실행 |
| **목적** | 상태 폴링(Polling), 모니터링, 알림 | 복잡한 문제 해결, 테스트 통과, 코드 리팩토링 완료 |
| **비유** | 매시간 순찰을 도는 **"경비원"** | 목표를 달성할 때까지 퇴근하지 않는 **"해결사"** |
