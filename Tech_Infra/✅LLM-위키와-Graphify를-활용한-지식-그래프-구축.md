# 🕸️ LLM 위키와 Graphify를 활용한 지식 그래프 구축

## 1. 배경 및 필요성
- LLM 위키에 데이터가 방대해지면 단순 텍스트 서치나 쿼리로는 비효율성이 발생할 수 있음.
- 문서 간의 연결 관계를 그래프 데이터베이스로 구축하여 AI(클로드 코드)가 전체 문서의 시맨틱 맥락을 참조할 수 있도록 함.

## 2. Graphify 기능 및 활용
- **역할**: 옵시디언 내 마크다운 문서를 분석하여 그래프 정보(`graph.json`)와 시각화 리포트(`graph.html`)를 생성.
- **설치**: Python 환경에서 라이브러리 설치 필요.
- **실행**: 클로드 코드에서 `graphify 위키폴더` 형식으로 명령을 실행하여 위키 폴더 전체를 그래프 형태로 변환 및 저장.

## 3. 그래프 기반 쿼리 (Graph RAG)
- 일반적인 텍스트 검색(쿼리) 대신, 생성된 그래프 정보를 기반으로 클로드 코드에게 질문을 던질 수 있음.
- 여러 노드와 엣지를 탐색하여 추상화 수준이 높은 질문(예: 특정 두 개념 간의 근본적인 차이점이나 상관성)에도 유의미한 답변을 제공.
- 데이터 갱신 시 업데이트 명령으로 변경된 부분만 재추출하여 그래프를 효율적으로 최신화.
