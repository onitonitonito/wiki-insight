Operations 06 / 10
핵심 워크플로우 — 위키를 살아있게 만드는 법
3가지 오퍼레이션과 도구 비교
Ingest — 자료 투입
새 원본을 Raw-Data에 추가 → AI가 분석하여 Wiki-Pages 생성. 자료 1개 → 15+ 페이지 동시 업데이트.
Query — 지식 탐험
위키에 질문 → AI가 기존 정리된 지식망에서 정확한 답변을 종합. 훌륭한 답변은 새 페이지로 자산화.
Lint — 위키 건강검진
주기적으로 AI가 고아 문서, 모순, 오래된 정보를 진단·치유하는 위키 유지보수 워크플로우.
도구 비교 원본 자료
"Obsidian = 뷰어, Claude Code = 편집자, Wiki = 내 지식 나무위키"

Conclusion 09 / 10
The New Paradigm
우리는 지식을
'검색'하지 않는다.
'탐험'하고 '결정'한다.
AI가 구축한 거대한 지식망 위에서, 인간은 더 이상 장부 관리자가 아닙니다.
우리는 그 위에서 탐험하고, 질문하고, 결정하는 지휘관이 됩니다.
"유지 관리 부담이 가치보다 빠르게 커지기 때문에 인간은 위키를 포기합니다. LLM은 지루해하지 않으며, 상호 참조를 업데이트하는 것을 잊지 않으며, 한 번에 15개의 파일을 처리할 수 있습니다."
— Andrej Karpathy, LLM Wiki Manifesto 2026
